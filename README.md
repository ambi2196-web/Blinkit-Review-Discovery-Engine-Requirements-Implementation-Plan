# Blinkit Review Discovery Engine

Scrapes public user feedback about Blinkit, tags it with an AI pipeline, and
surfaces quantified insights about why users don't explore new categories.

## Status

Phases 0-7 built. Reddit scraping is skipped (no PRAW app credentials) - Play
Store and App Store are the active sources, 6,000 reviews scraped, 2,258
survive cleaning. AI tagging runs on **Groq** `llama-3.3-70b-versatile`
(not Anthropic) - free tier caps at 100k tokens/day, so tagging fills in
incrementally over several days; just re-run `pipeline/tagger.py` (it's
idempotent). The dashboard and aggregates always reflect whatever's tagged
so far.

Phase 5 (validation) needs a human to hand-label `data/exports/holdout.csv`
(the `human_categories_mentioned` / `human_barrier_type` columns) before
`pipeline/validation.py --score` can run - by design, no model fills these in.

## Setup

```bash
python -m venv .venv
.venv\Scripts\activate        # Windows
pip install -r requirements.txt
copy .env.example .env        # then fill in GROQ_API_KEY yourself (don't paste it anywhere else)
```

## Run

```bash
python run_pipeline.py --init          # create data/reviews.db with empty schema (one-time)
python scrapers/play_store.py          # scrape Play Store reviews
python scrapers/app_store.py           # scrape App Store reviews
python pipeline/clean.py               # dedupe, filter, refine lang
python pipeline/tagger.py              # tag via Groq (re-run daily until quota-complete)
python pipeline/validation.py --sample # sample 50-review holdout (hand-label it, then --score)
python pipeline/aggregate.py           # build CSVs + insights.md from whatever's tagged
streamlit run app.py                   # local dashboard
```

## Deploy (Streamlit Community Cloud)

1. Push this repo to a public GitHub repo (`reviews.db` is committed - it's
   the dataset - `.env` is gitignored and never committed).
2. Go to [share.streamlit.io](https://share.streamlit.io), sign in with
   GitHub, click "New app", pick this repo/branch, set the main file to
   `app.py`.
3. In the app's Settings -> Secrets, add:
   ```toml
   GROQ_API_KEY = "your_key_here"
   ```
4. Deploy. Test the "Try It Live" tab on the deployed URL to confirm the
   live-demo call works in production.

## Repo structure

```
app.py                  # Streamlit dashboard (Phase 7)
scrapers/
    play_store.py       # Google Play scraper (Phase 1)
    app_store.py         # App Store RSS scraper (Phase 2)
pipeline/
    clean.py            # dedupe, lang detect, spam filter (Phase 3)
    tagger.py            # Groq batch tagging (Phase 4)
    validation.py         # AI-vs-human holdout validation (Phase 5)
    aggregate.py          # barrier x category matrix, insights synthesis (Phase 6)
prompts/
    tagging_prompt.md    # tagging system prompt + taxonomies + few-shot examples
data/
    reviews.db           # SQLite store (reviews, tags) - committed, it's the dataset
    exports/             # CSVs, dropped-review audit log, holdout sample
insights.md              # 10-bullet synthesis (auto-regenerated by aggregate.py)
validation_report.md     # written by validation.py --score, once holdout is hand-labeled
run_pipeline.py          # CLI: --init creates the empty schema
```
